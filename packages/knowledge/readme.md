## 实现思路

- 三种
- 普通知识库，有实体文件
  > 文件一般比较大,切割要保存多份,可编辑
- 字典型,无实体文件
  > 只读,有资源文件
  > 因为文件太多了,所以没法搞
- 图谱知识库,普通知识库升级版

  > 1有普通数据库2.单独一个图谱数据库存关系
  > 先处理普通数据库项,处理完成后处理图谱
  > 处理失败还需要回退
  > 字典数据库也可以改成图谱型?

- 不提供嵌入方法,只提供接口

- 实现的是集合,分为初始化集合和更新集合

- 图谱只是的chunksize不能改,因为如果改了,就会出现每次插入,节点分析的不同,
  > 如果要相同,就意味着n倍的时间

# 图处理操作

## 拆分节点

- A=>B,C
- 只涉及到删除A节点,删除A节点的边和将原来A节点/边改名为B,C然后再更新
- 还是涉及到词嵌入,因为名字变了,那么嵌入的内容就会发生变化,

## 合并节点

- B,C=>A
- 删除BC 创建A
- 需要把BC的东西都查出来,然后改名为A
- 需要词嵌入
